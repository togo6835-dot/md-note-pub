指数関数 ( \exp ) の数値不安定性まとめ

1. 何が問題か

数値計算では、指数関数

[ \exp(x) ]

が非常に急激に増減するため、( x ) が大きすぎたり小さすぎたりすると、計算機で正しく扱えなくなる。

overflow

大きすぎて表現できない状態。
たとえば倍精度では、おおよそ

[ x \gtrsim 709.78 ]

で

[ \exp(x) ]

は有限値として表現できず、inf になりうる。

underflow

小さすぎて表現できない状態。
たとえば

[ \exp(-1000) ]

のような値は、実際には極小の正数だが、計算機では 0 として扱われることがある。

NaN

overflow や underflow の結果として、

[ \frac{\infty}{\infty},\qquad \frac{0}{0},\qquad \infty-\infty ]

のような不定形が現れると、NaN が発生する。

2. 倍精度での基本的な目安

IEEE 754 の倍精度浮動小数点数では、だいたい次を覚えておくとよい。

[ \text{最大有限値} \approx 1.8 \times 10^{308} ]

[ \text{最小正規化正数} \approx 2.2 \times 10^{-308} ]

[ \varepsilon_{\mathrm{mach}} \approx 2.2 \times 10^{-16} ]

[ \exp(x)\text{ は }x \gtrsim 709.78\text{ で危険} ]

したがって、( \exp ) をそのまま計算する実装は非常に壊れやすい。

3. 基本方針

本質は一つである。

[ \text{「大きい } \exp \text{ を直接作らない」} ]

そのために、次のような工夫を行う。

  1. 対数のまま持つ
  2. 最大値や最小値を引く
  3. 和は log-sum-exp を使う
  4. 比は差で計算する
  5. 必要なら場合分けして指数の中身を非正にする

4. 最大値を引く方法

たとえば

[ w_i = \exp(a_i) ]

という重みがあるとする。
これをそのまま計算すると、( a_i ) が大きいと overflow する。

そこで

[ m = \max_i a_i ]

を取って

[ w_i = \exp(a_i-m)\exp(m) ]

と書く。
正規化にしか使わないなら共通因子 ( \exp(m) ) は消えるので、

[ \tilde w_i = \exp(a_i-m) ]

だけを計算すればよい。

このとき

[ a_i-m \le 0 ]

なので、指数の中身は常に非正となり、安全になる。

5. softmax の安定化

softmax は

[ p_i = \frac{\exp(a_i)}{\sum_j \exp(a_j)} ]

で定義されるが、これは素朴に計算してはいけない。

安定な形は

[ m=\max_j a_j ]

として

[ p_i=\frac{\exp(a_i-m)}{\sum_j \exp(a_j-m)} ]

である。

これにより、分子・分母に現れる指数の中身がすべて非正になる。

6. log-sum-exp trick

和

[ \sum_i \exp(a_i) ]

そのものを計算するのも危険である。
このときは

[ \mathrm{LSE}(a_1,\dots,a_N)= \log\left(\sum_{i=1}^N \exp(a_i)\right) ]

を直接ではなく、

[ m=\max_i a_i ]

を用いて

[ \log\left(\sum_i \exp(a_i)\right)= m+\log\left(\sum_i \exp(a_i-m)\right) ]

と計算する。

これが log-sum-exp trick である。

なぜ安全か

すべての ( a_i-m \le 0 ) なので

[ \exp(a_i-m)\le 1 ]

であり、巨大な指数が出ない。

7. 確率の正規化との関係

log probability

[ x_i = \log p_i ]

が与えられたとき、正規化された確率は

[ p_i = \frac{\exp(x_i)}{\sum_n \exp(x_n)} ]

である。

これを安全に計算するには

[ \log p_i= x_i-\log\left(\sum_n \exp(x_n)\right) ]

とし、さらに log-sum-exp を用いて

[ \log\left(\sum_n \exp(x_n)\right)= m+\log\left(\sum_n \exp(x_n-m)\right), \qquad m=\max_n x_n ]

と計算する。

8. ( e^{-S} ) 型の重みに対する適用

論文や統計力学、経路積分では

[ w_i = e^{-S_i} ]

のような重みが現れる。
ここで ( S_i ) が大きくばらつくと、やはり数値的に不安定になる。

このときは

[ S_{\min} = \min_i S_i ]

を取って

[ w_i = e^{-(S_i-S_{\min})}e^{-S_{\min}} ]

と書く。
正規化だけが必要なら共通因子 ( e^{-S_{\min}} ) は不要なので、

[ \tilde w_i = e^{-(S_i-S_{\min})} ]

だけ計算すればよい。

ここでは

[ S_i-S_{\min}\ge 0 ]

だから

[ -(S_i-S_{\min})\le 0 ]

であり、指数の中身が非正になる。

正規化重み

[ p_i= \frac{e^{-S_i}}{\sum_j e^{-S_j}}= \frac{e^{-(S_i-S_{\min})}}{\sum_j e^{-(S_j-S_{\min})}} ]

分配関数の対数

[ \log\left(\sum_i e^{-S_i}\right)= -S_{\min} + \log\left(\sum_i e^{-(S_i-S_{\min})}\right) ]

これは ( x_i=-S_i ) とおいた log-sum-exp trick そのものである。

9. 差だけ必要なら差で計算する

比

[ \frac{e^a}{e^b} ]

は、それぞれを別に指数化するより

[ e^{a-b} ]

と書く方が安全である。

同様に

[ \frac{e^{-S(\phi)}}{e^{-S(\chi)}} = e^{-(S(\phi)-S(\chi))} ]

である。
絶対値ではなく差が本質なら、最初から差で計算する。

10. Metropolis 型の受理率

たとえば

[ A=\min{1,e^{-\Delta S}} ]

を計算する場合、素朴に常に指数を計算する必要はない。
次のように分岐すればよい。

[ \Delta S \le 0 \Rightarrow A=1 ]

[ \Delta S > 0 \Rightarrow A=e^{-\Delta S} ]

これなら、実際に計算する指数の中身は常に非正である。

11. 深層学習でも同じことをしている

深層学習ライブラリでも、この種の数値安定化は標準である。

softmax / log-softmax

[ \log(\mathrm{softmax}(x)) ]

を素朴に

[ \log\left(\frac{e^{x_i}}{\sum_j e^{x_j}}\right) ]

と計算すると不安定なので、内部では最大値を引いた形や log-sum-exp が使われる。

cross entropy

softmax を明示的に計算してから log を取るのでなく、logits から直接、安定な形で loss を計算する。

sigmoid cross entropy

2値分類の損失では

[ x-xz+\log(1+\exp(-x)) ]

のような式が現れるが、( x<0 ) で (\exp(-x)) が危険になる。
そこで

[ \max(x,0)-xz+\log(1+\exp(-|x|)) ]

のように、指数の中身が非正になる等価変形を用いる。

12. TensorFlow の sigmoid_cross_entropy_with_logits の意味

sigmoid_cross_entropy_with_logits では、損失が最終的に

[ x-xz+\log(1+\exp(-x)) ]

と書ける。
しかし ( x<0 ) だと (-x>0) なので、(\exp(-x)) が大きくなり危険である。

そこで実装では

[ \max(x,0)-xz+\log(1+\exp(-|x|)) ]

を使う。

ここで

[ -|x|\le 0 ]

なので、指数関数は安全である。

13. PyTorch でも同様

PyTorch でも

  • log_softmax
  • cross_entropy
  • BCEWithLogitsLoss

などが、内部で安定化された形を用いる。

つまり、

[ \exp ]

をむき出しで計算するより、

  • log_softmax
  • logsumexp
  • cross_entropy
  • binary_cross_entropy_with_logits

のような関数を使うのが基本である。

14. 直感的な理解

本質は、数式の上では有限でも、計算機では途中計算が壊れるという点にある。

たとえば

[ \frac{\exp(1000)}{\exp(1000)+\exp(999)} ]

は数学的には有限だが、計算機はまず ( \exp(1000) ) を表現できない。
そこで

[ \frac{\exp(0)}{\exp(0)+\exp(-1)} ]

のように同値変形してから計算する必要がある。

15. 実務的な指針

指数関数が現れたら、次を確認するとよい。

和の中に ( \exp ) がある

[ \sum_i \exp(a_i) ] なら log-sum-exp を使う。

正規化重み

[ \frac{\exp(a_i)}{\sum_j \exp(a_j)} ] なら最大値を引く。

( e^{-S} ) 型

[ e^{-S_i} ] なら ( S_{\min} ) を引いて [ e^{-(S_i-S_{\min})} ] にする。

比

[ \frac{e^a}{e^b} ] なら [ e^{a-b} ] にする。

受理率

[ \min{1,e^{-\Delta S}} ] なら符号で分岐する。

できれば log のまま保持

確率や重みは、可能なら [ p ] ではなく [ \log p ] を保持する。

16. 結論

指数関数が入る数値計算では、次が中心原理である。

[ \text{大きい指数を直接作らない} ]

そのための代表的手法は

  1. 最大値・最小値を引く
  2. log-sum-exp を使う
  3. 差で計算する
  4. log のまま持つ
  5. 必要に応じて安定な等価変形を使う

である。

特に

[ e^{-S} ]

型の重みでは

[ e^{-S_i} \quad\longrightarrow\quad e^{-(S_i-S_{\min})} ]

という変形が基本であり、これは深層学習で使われる softmax や BCE の数値安定化と本質的に同じ発想である。
